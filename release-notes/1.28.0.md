# Forge Method 1.28.0

Guidance audit hardening for runtime-builder routing, stale plugin repair, and script quality.

## Highlights

- Routed method/runtime audit requests, dead-code concerns, misleading agent docs, and human-guided experience questions to `builder-flow` / `runtime-builder`.
- Added a transcript fixture for runtime audit guidance so this route cannot silently regress to `operate-support`.
- Updated the internal Guidance Engine benchmark with runtime audit expectations.
- Added `doctor` repair commands for stale or broken local plugin installs.
- Updated quickstart guidance to follow the printed `Repair:` command when `doctor` is not ready.
- Audited scripts with Ruff, Vulture, Radon, ShellCheck, PSScriptAnalyzer, PowerShell parser checks, `bash -n`, and `py_compile`.
- Fixed shell and PowerShell script warnings around unused variables, literal `done`, empty catch blocks, and explicit `-Python` handling.
- Recorded the script audit, complexity hotspots, market hook/tracing references, and isolated experiment plan in a durable Forge artifact.
