# Forge Method 1.34.0

Forge Method Core v1.34.0 ships guided team collaboration, Product Areas, and repo split planning:

- add `Product Area` as the product-facing ownership boundary, while keeping `Forge Module` reserved for runtime/workflow packaging
- add `Root Integrator Project`, `Team Operating Model`, `Collaboration Handoff`, `Repo Split`, and `Standalone Method Project` to the canonical collaboration model
- add guided workflows for team operating model, product area mapping, trunk-based planning, collaboration handoff, and repo split planning
- route team, GitHub org, trunk-based, CODEOWNERS, monorepo, multi-repo, ownership, handoff, modularization, and standalone repo requests through collaboration guidance before build work
- default to monorepo-first Root Integrator Projects, with repo split only after owner, contract, validation, release boundary, and integration cost are explicit
- define split behavior so an extracted Product Area receives compact standalone `.forge-method/` context and the root integrator keeps integration evidence
- extend story, sprint, status, and build work templates with optional Product Area, owner, branch, PR, dependency, and handoff fields without adding solo-project overhead
