# Forge Method Core 2.0.5

Hotfix for route authority contamination in checkpoint and handoff memory.

## What Changed

- `checkpoint --next-action` writes a non-authoritative continuation suggestion by default.
- `handoff --next-action` follows the same memory-only default.
- Use `--update-state` when a workflow or human decision intentionally authorizes a route change.
- Fleet `requests apply` ignores legacy checkpoint/handoff route mutations without `state_update_authorized`.
- Checkpoint and handoff artifacts now label route authority as `state`, `checkpoint-suggestion`, or `handoff-suggestion`.

## Why It Matters

Forge should not get stuck following an agent's own invented prototype continuation just because that continuation was recorded in a checkpoint. Durable memory now preserves the suggestion without pretending it is the official next step.
