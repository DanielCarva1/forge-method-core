# Forge Method 1.31.1

Public install route hotfix.

## Highlights

- Installed public Forge packages no longer expose committed Forge core state as a normal project.
- Normal users who start Forge inside the installed package are guided to open or create a separate project workspace.
- Core-runtime work now requires explicit maintainer intent via `FORGE_METHOD_CORE_DEV=1` or local `.forge-method/core-dev.local`, plus `--allow-runtime-state`.
- Plugin manifest detection now tolerates UTF-8 BOM, which prevents Windows-authored manifests from bypassing installed-package routing protections.
