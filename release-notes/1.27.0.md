# Forge Method 1.27.0

Native Guidance Engine for human-intent routing.

## Highlights

- Added Guidance Engine behind `guide --question --json`.
- Added workflow catalog metadata for phase, required/optional status, follow-up routing, outputs, aliases, and facilitation packs.
- Added human facilitation packs for discovery, game brief, brainstorming, problem solving, design thinking, test strategy, runtime builder, and correct-course.
- Added guided depth workflow families for game lifecycle, test architecture, builder utility, and document utility work.
- Added route-aware selection for game story creation, game UX/PRD, game test framework/automation/E2E, test framework/CI/ATDD/review/NFR/traceability, workflow/agent analysis, skill conversion, doc indexing/sharding, editorial/edge-case review, and spec distillation.
- Added artifact templates for guided depth families and validation for optional workflow catalog template references.
- Added domain examples to guided-depth facilitation packs and transition commands for entering narrow guided-depth workflows from `guide --question --json`.
- Added `teach-testing` for applied testing education requests before test strategy or framework setup.
- Made workflow validation check packaged module workflow ids, catalog aliases, metadata, and facilitation pack references.
- Classified correction, confusion, brainstorm, research, creative, game, quality, builder, mechanical build, support, and evolution intent.
- Made ready projects route critique or new intent to `6-evolve` or `correct-course` instead of repeating stale release work.
- Fixed mixed method-parity requests so runtime-builder context outranks incidental game/test keywords.
- Updated Hot Start to run Guidance Engine after resume when the latest user message contains substantive intent.
- Added transcript fixtures and an internal benchmark artifact for guided human routing.
- Kept the emergency reload command and `$forge-reload` skill in this delivery batch.
