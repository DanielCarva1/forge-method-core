# Forge Method 1.31.2

Guided research drift hotfix.

This patch keeps strategic app, stack, interface-language, and codebase-standard conversations in guided research instead of letting builder/runtime keywords push the agent into generic automation too early.

Highlights:

- Stack/language/interface decisions with signals like long research, study, iteration, native UI options, editor references, agent self-modification references, or Rust codebase standards now route to technical-feasibility research.
- Ready projects preserve the `6-evolve` route when research opens an evolution cycle.
- Performance wording such as "mais rapido e performativo" no longer gets mistaken for a rushed fast-path request.
- Parity replay now includes a standalone-app transcript regression fixture.

Validation:

- `python -m unittest discover -s tests`
- `powershell -ExecutionPolicy Bypass -File .\scripts\verify-fast.ps1 -Test tests.test_runtime.RuntimeTests.test_standalone_stack_conversation_stays_in_research_guidance`
- `powershell -ExecutionPolicy Bypass -File .\scripts\smoke-runtime.ps1`
