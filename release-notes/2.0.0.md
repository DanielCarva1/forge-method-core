# Forge Method Core 2.0.0 — Flock Coordination

**Released:** 2026-06-23

The coordination protocol for human + agent flocks. Multiple humans, each with multiple agents, working on one repo without chaos.

## Highlights

- **G1/G2/G3 blockers fixed** — state clobbering, handoff clobbering, no agent identity. All three multi-agent blockers from the v1 audit are resolved.
- **Lane claims with TTL** — agents claim lanes before writing; two agents never edit the same lane simultaneously. Claims auto-expire.
- **`forge-commit` wrapper** — stages only files in your claimed lanes. No more `git add -A` cross-contamination.
- **Requests queue** — workers emit requests to `requests.ndjson`; the driver polls and applies with version-checked writes.
- **Guidance safety extended** — multi-agent anti-patterns (write-without-version, act-outside-lane) + autonomy anti-patterns (approve-during-autopilot).
- **Partner experience** — research-always-on affordance in all 34 facilitation packs, grill-as-default at every decision-close point.
- **Integration gate** — `tsc --noEmit` + smoke scripts in the gate (GAP-5 from POC).
- **Council standup** — fleet coordination summary across all lanes.
- **AGENTS.md emitter** — generates draft AGENTS.md for Claude Code / OpenCode (human-approval gated per ETH Zurich findings).
- **Evolve routing fixed** — guidance engine now correctly routes evolve-phase build-intent to discovery (not builder).

## New commands

| Command | What |
|---|---|
| `claim <lane>` | Claim a lane for multi-agent coordination |
| `unclaim <lane>` | Release a lane claim |
| `heartbeat <lane>` | Renew a claim (reset TTL) |
| `lanes` | Show all lane claim statuses |
| `forge-commit -m "msg"` | Git commit that stages only claimed-lane files |
| `requests poll` | Driver: read pending worker requests |
| `requests apply <index>` | Driver: apply a request with version check |
| `council standup` | Fleet coordination summary |
| `contract create --name ... --lane ...` | Typed agent-contract artifact |
| `spawn --lane ... --task ...` | Emit a spawn directive |
| `emit-agents-md --runtime ...` | Generate AGENTS.md draft |
| `handoff --no-update-state` | Force append-only handoff (fleet mode) |
| `transition --expected-version N` | Version-checked phase transition |
