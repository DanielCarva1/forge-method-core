# Forge Method 1.31.0

Parity closure and runtime utilities.

## Highlights

- Remaining parity audit utility surfaces are now translated into Forge-native, opt-in contracts: isolated eval runner, hook/event plan, and API/browser utility.
- Guidance Engine can route explicit eval runner, hook/event, and reusable API/browser utility requests to dedicated workflows with facilitation packs and compact artifact templates.
- Utility scripts are explicit and disabled by default: eval execution supports dry-run/local/container decisions, and hook dispatch requires an opt-in command.
- Replay coverage now protects these routes alongside the broader guidance fixture set.
- Release/version skepticism now routes to release-readiness so agents verify repo, tag, version metadata, and install evidence before claiming publication status.
