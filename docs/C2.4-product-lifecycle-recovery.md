# C2.4 product lifecycle recovery corpus

C2.4 is a closed, host-neutral **source corpus** for the lifecycle surface in
[`product-lifecycle.md`](product-lifecycle.md). Its cases describe inputs,
interruption boundaries, expected preservation, and observations that source
tests may consume. They do not execute a lifecycle operation and are not
stabilization evidence.

The machine-readable corpus is
[`C2.4-product-lifecycle-recovery-corpus.yaml`](fixtures/product-lifecycle-recovery-v0/C2.4-product-lifecycle-recovery-corpus.yaml).
Every case requires the stated explicit install root. No root may be inferred
from a consumer repository, working directory, Project Link, or Forge sidecar.

## Source cases

| Case | Required source assertion |
| --- | --- |
| Partial setup | An ownership marker without complete valid lifecycle state is not an adopted installation. Status and doctor expose it as incomplete or degraded; setup must not overwrite unknown retained data. |
| Stale host configuration | A missing, changed, or unsafe retained configuration is an exact diagnostic observation only. It neither selects nor supports a host. |
| Interrupted update | A staged or partial candidate generation cannot become active. The prior verified active generation remains active and a later update must re-verify its input. |
| Downgrade | Update rejects a release below the active semantic version without changing active or previous generation identity. |
| Wrapper mismatch | A changed wrapper digest is reported and preserved; no replacement, activation, or inferred host action follows. |
| Retained-state uninstall | Uninstall removes only matching product inventory while retaining unknown, modified, sidecar, anchor, backup, receipt, and external-key material. |
| Backup interruption | An incomplete backup artifact is not lifecycle state, does not authorize restore, and cannot cause external-key inclusion. |
| Replacement-machine restore | Candidate recovery material may be observed and validated, but cannot activate a generation, select a host, or recreate excluded authority. |

## Authority and custody boundary

Corpus documents, inputs, expected results, and observations are candidate data
only. Deserializing them grants no claim, mutation, phase-transition, release,
signing, trust, admission, install, activation, lifecycle, protected-anchor,
private-key, or host-selection authority. `selected_host` is always `null`.

Consumer projects, Project Links, Forge sidecars, operator anchors, backups,
restore receipts, public external-broker registries, signing keys, trust roots,
and external private broker keys remain outside lifecycle custody. In
particular, private external broker keys never enter Forge state, Forge backups,
fixtures, or replacement-machine recovery material.

## Evidence boundary

The corpus is authored source input and does not claim execution. Rust source
tests, interrupted-operation and mixed-version execution, crash/failure
injection, backup and replacement-machine restore runs, stress/fuzz/benchmark
work, MSRV/platform/target matrices, hosted CI, publication, real-host checks,
and field evidence remain unexecuted stabilization work.
