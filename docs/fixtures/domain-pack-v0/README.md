# Domain Pack v0 neutral fixture corpus

This corpus demonstrates P6a's **read-only, candidate-only** composition contract. It is generic on purpose; the real game-development pack remains P6d work.

## Scenarios

- `requests/neutral-two-pack.yaml` composes `forge.fixture/foundation@1.0.0` and its dependent `forge.fixture/assurance-extension@1.1.0` against one sealed, empty core fixture bundle.
- `requests/neutral-extension-removed.yaml` removes only the extension candidate while preserving the independent project requirement for `sample.extension.domain.independent-assurance`. Its derived projection must therefore be blocked by an explicit `missing_domain` gap.
- `requirements/neutral-delivery.yaml` is the standalone source for those persistent requirements; candidate removal never edits it.

The content documents cover policy, obligation, claim, advisory playbook, hazard, lifecycle, evaluator, fixture, capability, and adapter declarations. Evaluators, adapters, capabilities, provenance, and prose are declarations or untrusted data only. Nothing here installs, activates, executes, trusts, or mutates a Domain Pack or Forge Core.

## Digest convention

All wire digests use `sha256:<lowercase-hex>`. `raw_sha256` binds the exact UTF-8 bytes in the referenced file. `canonical_sha256` is SHA-256 over the RFC 8785/JCS-compatible JSON serialization of the parsed document (`sort_keys`, compact separators, and no floating-point values in this corpus). The manifest's `required_content_digest` binds the foundation document's canonical digest.

The composition requests embed the same typed manifest/content values so composition needs no filesystem discovery, registry, network access, or ambient dependency selection. Expected projection files are generated by the pure Forge composer, not authored as authority claims.
