# Expansion Roadmap

Forge Method expands through one public Codex entrypoint, `$forge-method`, while the runtime helper exposes deterministic subcommands for agents.

## Product Layers

Human Experience is the user-facing layer: guide output, agent personalities, onboarding, tutorials, live council discussions, and explanations.

Agent Runtime is the agent-facing layer: skills, workflows, manifests, state files, evals, gates, artifacts, and recovery files. Runtime documents stay compact and structured.

## Tracks

Tracks route the user into the right operating shape:

- `quick-flow`
- `standard-product`
- `enterprise`
- `creative-studio`
- `game-studio`
- `runtime-builder`
- `test-architect`
- `launch-ops`

Tracks do not create new public skills. They are runtime decisions behind `$forge-method`.

## Agent Council

Agent Council is optional. It is useful for high-risk, taste-heavy, or strategically important decisions. The live discussion is shown to the human. The durable memory is a compact Council Decision Artifact with participants, agreements, disagreements, risks, recommendation, decision, and next action.
