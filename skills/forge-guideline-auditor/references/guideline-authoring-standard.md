# Guideline Authoring Standard

Use this when creating or reviewing a guideline.

## Required Sections

- Purpose
- Source Inputs
- Applies To
- Human Promise
- Agent Rule
- Machine Contract
- Allowed
- Forbidden
- Examples
- Acceptance Evidence
- Checks
- Update Triggers
- Work Order Bridge

## Rules

- Use MUST, SHOULD, MAY, and MUST NOT consistently.
- Translate external sources into local Forge rules.
- Separate human trust from agent mechanics and machine checks.
- Name forbidden behavior explicitly.
- Say when the guideline is speculative or draft.
- Prefer observable evidence over quality adjectives.

## Evidence Examples

- command output
- screenshot
- fixture diff
- generated file with source list and timestamp
- unit test result
- policy table
- before/after documentation diff
- release note entry
- rollback note

## Bad Evidence

- looks good
- follows best practices
- clean code
- should work
- the model understands it
