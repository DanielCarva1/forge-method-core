# Project Instructions

This project uses Forge Method.

Runtime state lives in `.forge-method/`. Before continuing work, inspect:

- `.forge-method/state.yaml`
- `.forge-method/sprint.yaml`
- `.forge-method/context/latest-checkpoint.md`
- the active story file
- recent evidence
- the current context pack when present

Do not infer phase, story, or readiness from chat history. Use state files.

Before marking work done:

- satisfy acceptance criteria
- run or document checks
- write evidence
- write a checkpoint for meaningful progress
- update story status
- update runtime state

Agent-facing workflow docs should stay compact state machines.
